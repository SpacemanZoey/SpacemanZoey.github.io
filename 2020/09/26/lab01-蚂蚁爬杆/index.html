lab01-蚂蚁爬杆

0. 题目详情

有一根300 厘米的细木杆，在第30 厘米、80 厘米、110 厘米、160 厘米、250厘米这五个位置上各有一只蚂蚁。木杆很细，不能同时通过两只蚂蚁。

开始时，蚂蚁的头朝左还是朝右是任意的，它们只会朝前走或调头，但不会后退。当任意两只蚂蚁碰头时，两只蚂蚁会同时调头朝相反方向走。假设蚂蚁们每秒钟可以走5 厘米的距离。

注：不允许穿越对方身体继续直行

请编写一个程序，计算各种可能情形下所有蚂蚁都离开木杆的最小时间和最大时间。

1. 思路概要

在蚂蚁类中设置position这一属性，通过一个函数f(x)对position进行加减来实现蚂蚁的运动，当蚂蚁的position满足下式：
$$
position \leq 0 \quad||\quad position \geq 300
$$
说明蚂蚁已经离开木杆，此时我们将蚂蚁类中isDown的属性设置为true，并将WoodenPole类中表示离开蚂蚁数量的属性numOfDown++，当numOfDown == antNum所有的蚂蚁已经全部离开木棒，游戏结束。

此外，我们考虑到需要枚举所有蚂蚁的初始面朝方向的情况，所以我们将蚂蚁的面向direction设置boolean型，面朝右则为true，反之则为false，通过二进制的移位来设置每一只蚂蚁的面向情况，使用一个二维数组来记录这些值。

而后，我们需要考虑计算蚂蚁离开木棒所用时间，我们使用了在CreepingGame类中设置了thisTime来记录蚂蚁所用的时间，最后取最大最小值输出。

2. 具体思考过程

（一）基本功能

（1）蚂蚁爬行；（2）蚂蚁转方向；（3）判断蚂蚁相遇；（4）判断蚂蚁爬下木棒；（5）判断游戏结束；（6）计算游戏进行时间；（7）计算蚂蚁面向的所有可能；（8）遍历所有的面向可能；（9）找出所有可能中的最大时间和最小时间；

（二）对象设置

（1）Ant；

（2）CreepingGame（用于执行一整次的爬行活动）；

（3）PlayRoom(用于初始化和遍历所有可能，求出最大最小值)

（4）WoodenPole（用于判断蚂蚁是否在杆上）

（三）部分功能的实现逻辑的思考过程及困难克服

（1）蚂蚁爬行；

初始思考方案是position + = speed*faceDirection 但是这样会导致蚂蚁的行进过程是离散的，换言之蚂蚁的行进就像是在棋盘上一格一格走。

这样会导致两个问题：

一是相遇地点的误差（不带小数点）会导致时间计算的误差；

二是假设1号蚂蚁在20格向后走，2号蚂蚁在21格向前走，那么正确的情况在是蚂蚁在20.5的位置相遇然后转方向，但是在计算机中操作是有执行顺序的。

这个逻辑的操作如下：①1号蚂蚁走一格 ②2号蚂蚁走一格 ③检查是不是有相遇

如果顺序是①②③，那么结果就是蚂蚁互相错过，不会被认为碰头了。

（3）蚂蚁转方向

我们将代表蚂蚁朝向的属性direction设置为boolean类型，蚂蚁调转方向则为direction = !direction。

（4）判断蚂蚁相遇；

针对离散型的特定修改：

当相邻的两只蚂蚁朝向相反的时候，position[i] == position[i+1]时，蚂蚁相遇，此外，我们考虑蚂蚁撞头的情况position[i] > position[i+1]，那么此时，蚂蚁需要往回走一步

（5）判断蚂蚁爬下木棒；

Position<0 或者 position > 300，这里的数字可以根据木棒的长度调整。

（6）判断游戏结束；

我们在WoodenPole类型中设计了一个int类型的numOfDown属性，这一属性代表了已经下杆的蚂蚁，当numOfDown == antNum时，所有蚂蚁均已下杆。

（7）计算游戏进行时间；

详细一点的方案是在开始和结束的时候返回系统时间然后减一减，但是老师讲这个是简化版的，那么就自己定义一个计时器int time = 0 每次全部蚂蚁走完一次之后（即每个蚂蚁都完成一次position + = velocity * thisTime）后thisTime++ 算是完成一个时钟

（8）计算蚂蚁面向的所有可能；

利用二进制数的特性做逻辑操作，假设有4只蚂蚁，那么0000 0001 0010 ~1111 总共16个数字（1-16）就代表全16种面向的可能
